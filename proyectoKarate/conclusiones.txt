						==============================================
						Proyecto API Testing – PetStore con Karate 
						Framework + Gradle
						==============================================

📘 Objetivo del ejercicio:

Automatizar pruebas funcionales de la API de PetStore utilizando Karate Framework, parametrizando los datos desde archivos JSON para validar operaciones CRUD de mascotas (crear, consultar, actualizar y filtrar por estado) y documentar los resultados mediante reportes Cucumber.

----------------------------------------------
📊 Resumen de resultados:

- Pruebas API completadas exitosamente
- Automatización parametrizada
- Reportes generados y trazabilidad

----------------------------------------------

💡 Conclusiones y Hallazgos

* El enfoque BDD en Karate permite estructurar las pruebas de APIs en pasos (GIVEN, WHEN, THEN), segmentando el flujo de validación y facilitando la comprensión del comportamiento esperado.

* El enfoque BDD en Karate genera reportes intuitivos que permiten identificar con precisión en qué paso del escenario se produjo un fallo y análisis de errores.

* Karate permite validar la integración de APIs y el manejo de errores en el backend, lo que resulta especialmente útil para pruebas confiables de APIs públicas.

* Durante la automatización del sitio https://petstore.swagger.io/, se evidenció inestabilidad en las API que provocaba fallos en las pruebas.
 
* Para solucionarlo, se implementaron validaciones de tipo de datos y IDs definidos manualmente.
